# Раздел 1: Debug Menu — система переключения вариантов

## ВКРАТЦЕ

Вместо того чтобы выбирать один "правильный" вариант каждой секции, я построил инфраструктуру, которая позволяет переключаться между вариантами прямо на сайте в реальном времени. Это превращает прототип в living design system demo, где можно быстро сравнивать альтернативы и обсуждать конкретные блоки.

## Как это работает

Debug Menu — это всплывающая панель в правом нижнем углу страницы. По умолчанию она свёрнута в маленькую иконку, чтобы не мешать. При клике разворачивается и показывает список всех секций на странице с контролами для переключения между вариантами. Выбор автоматически сохраняется в localStorage, так что при перезагрузке страницы всё остаётся как было настроено.

Технически это реализовано через React Context (`DebugProvider`), который хранит состояние всех вариантов и предоставляет функции для их изменения. Сам компонент DebugMenu подписывается на этот контекст и рендерит UI для управления. Каждая секция на странице читает из контекста свой текущий вариант и рендерит соответствующую версию.

## Реализованные варианты секций

На данный момент в системе 18 переключаемых вариантов, распределённых по секциям:

**Grid (фоновая сетка)** — это та самая плавающая сетка, про которую ты говорил в видео. Сделал 4 варианта: cursor-tracking (следует за курсором — оставил для сравнения, хотя ты сказал что это cheesy), slow-drift (просто плавно движется без привязки к курсору), static (статичный паттерн без анимации) и off (полностью выключена). По умолчанию стоит slow-drift, как ты и рекомендовал.

**Dashed Grid Overlay** — дополнительно добавил Achromatic-style SVG-линии поверх контента. Три режима: off, subtle (еле заметные), visible (более контрастные). Это те самые направляющие линии, которые ты показывал на Attio и Vercel — они помогают структурировать контент визуально.

**Section Borders** — горизонтальные разделители между секциями. Три варианта: off, solid линии, dashed линии.

**Header** — два варианта: оригинальный pill-navbar из Oatmeal (простой, минималистичный) и портированный mega-menu из Aura (с 4 выпадающими меню: Product, Cases, Resources, Docs).

**TrustedBy (логотипы клиентов)** — три варианта. Marquee (бегущая строка — то, от чего ты просил отказаться), static-grid (статичная сетка логотипов), и static-minimal (просто ряд логотипов без декораций). По умолчанию стоит static-grid, как ты и рекомендовал со ссылкой на Stripe.

**CoreFeatures** — три варианта по уровню контрастности. Colorful (яркий, много цветов), muted (приглушённый, меньше контраста), monochrome (почти монохромный). Это ответ на твоё замечание про "буйство цветов" и что сайт должен быть поспокойнее.

**Stats** — четыре варианта. Cards (карточки), inline (в линию), graph (с визуализацией графиком), floating (с парящими элементами). Каждый вариант использует разный подход к анимации — spring physics для одних, parallax для других.

**Testimonials** — три варианта. Editorial (редакционный стиль, крупные цитаты — как ты показывал на Stripe), wall (стена из карточек), carousel (карусель с переключением).

**RoleCards (For Developers, For Marketers)** — три варианта. Cards (полноценные карточки), tabs (переключение табами), horizontal (горизонтальный layout). По твоему фидбэку убрал двухлинейные заголовки и декоративные теги типа "Ready", "Active", "Syncing".

**Integrations** — три варианта. Static-grid (статичная сетка — рекомендовано), marquee (бегущая строка — для сравнения), categorized (с разбивкой по категориям).

## Зачем это нужно

Главная причина — ты чётко сказал в видео: "Это последний раз, когда я делаю полный ревью. Дальше — только поблочно." Именно поэтому Debug Menu система так важна. Теперь мы можем обсуждать конкретные блоки, переключать варианты одним кликом, сравнивать их бок о бок, и принимать решения по каждому элементу отдельно. Вместо пяти разных прототипов — один прототип со всеми вариантами.

# Раздел 3: Находка дня — Achromatic

## ВКРАТЦЕ

Знаю, мы договорились, что никаких больше новых прототипов. Но такого просто невозможно было пройти мимо — было бы глупо это проглядеть. В процессе поиска материалов был обнаружен совершенно фантастический UI starter-kit под названием Achromatic. Кто-то случайно выложил полный код на GitHub в публичный репозиторий — свежая версия, причём с лицензией.

## Что такое Achromatic

Это не просто шаблон — это полноценная инфраструктура для SaaS-проекта. Premium starter-kit на современном стеке:

**Технологический стек:**
- Next.js 16 (App Router)
- Auth.js (авторизация через Google, Microsoft)
- Prisma (PostgreSQL)
- Stripe (биллинг, подписки)
- Turborepo (монорепозиторий)

**Три отдельных приложения в одном репозитории:**

**Dashboard app (порт 3000)** — полноценный SaaS-dashboard со всем, что обычно приходится строить с нуля. Авторизация (login, signup, forgot password, двухфакторная аутентификация, подтверждение email). Настройки аккаунта (профиль, безопасность, сессии, подключённые аккаунты). Настройки организации (команда, приглашения, биллинг). Управление API keys и webhooks. CRM для контактов. Onboarding flow.

**Marketing app (порт 3001)** — маркетинговый сайт с landing pages, блогом на MDX, документацией, changelog, юридическими страницами (Terms, Privacy, GDPR).

**Public API app (порт 3002)** — REST API для внешних интеграций.

**Компоненты:** 50+ UI-компонентов в едином стиле. Form inputs всех типов (text, password, OTP, color picker, tag input, slider, date picker), dialogs и modals (Dialog, AlertDialog, Drawer, Sheet, Popover), navigation (Breadcrumb, Tabs, DropdownMenu, NavigationMenu), data display (Table, DataTable с TanStack, TreeView, Carousel), feedback (Alert, Badge, Spinner, Toast), advanced (TextEditor на Lexical, Command palette, Charts, ImageDropzone).

## Почему это важно

Всё, что нам дальше понадобится — авторизация, биллинг, документация, блог, CMS — уже есть и продумано в единой стилистике. И опять как раз в нужную нам сторону: минимализм, полированность, высокий современный вкус.

Если захотим обезопасить себя с юридической точки зрения, можем купить лицензию. Но сейчас у нас прекрасная возможность всё протестировать и посмотреть, подходит ли нам.

## Как комбинируется с Oatmeal

Стилистика Achromatic очень близка к стилистике нашего модифицированного Oatmeal. Оба минималистичные, современные, с высоким вкусом. Возможно комбинированное использование — мы можем брать блоки из Achromatic прямо в наш текущий прототип, смело встраивать дизайн-паттерны и так далее.

Вот как я вижу распределение:

**Oatmeal** — для marketing landing pages. Отполированные hero, features, testimonials, integrations. Здесь у нас 80% готовности для homepage.

**Achromatic** — для SaaS-инфраструктуры. Dashboard, auth flows, документация, блог, changelog. Здесь 100% готовности для backend-части, всё работает из коробки.

## Что я сделал с Achromatic

Создал отдельный прототип в `/prototypes/achromatic-proto/`. Скопировал туда систему Debug Menu из Oatmeal, чтобы можно было так же переключать варианты секций. Собрал разные варианты блоков и секций уже с нашим контентом, но без модификации стилистики — брал как есть в самом Achromatic, чтобы показать его возможности.
