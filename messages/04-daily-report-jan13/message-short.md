## Напоминание о методологии

Чтобы было понятно, о чём речь. Мы договорились тестировать подходы системно, а не угадывать:

**Phase A** (завершён, развёрнут): прототип на чистом Tailwind CSS с 5 разными дизайн-системами. Можно переключаться между ними и сравнивать. Цель была проверить, какой визуальный стиль лучше работает. https://adapty-prototype.vercel.app

**Phase B** (сегодняшняя работа): тестирование UI-библиотек поверх выбранного стиля. Гипотеза была в том, что библиотеки ускорят разработку и улучшат качество. Для теста выбрал shadcn/ui как самый популярный и интегрированный с AI-инструментами вариант.

## Почему мы ещё тестируем инструменты

Важный момент, который хочу объяснить. Тестирование инструментов это не "мы всё ещё ищем" в плохом смысле. Это осознанный процесс, который экономит время в долгосрочной перспективе.

Представь, что мы бы просто взяли shadcn/ui и начали строить весь сайт. Через неделю работы стало бы понятно, что результат получается generic и не дотягивает до уровня Linear или Attio. Пришлось бы переделывать. Потерянное время, потерянная мотивация.

Вместо этого я потратил один день на системный тест. Построил два полноценных прототипа (14 секций и 10 секций), сравнил результат с Phase A, и теперь точно знаю: shadcn/ui не подходит для маркетинговых страниц.

Теперь у меня есть обоснованное решение вместо догадки. И ещё одна гипотеза для проверки, которая может ускорить разработку ещё сильнее.

## Главный вывод дня

Чистый shadcn/ui не подходит для маркетинговых страниц. Это отличная библиотека для дашбордов, но для landing pages, где важен каждый пиксель, базовые компоненты добавляют лишний слой абстракции без прироста качества.

Результат: Phase A на чистом Tailwind дал качество 7/10. Phase B на чистом shadcn дотянул только до 4/10.

Но это не значит, что мы полностью отказываемся от shadcn-экосистемы. Есть ещё shadcn-based шаблоны и component libraries (типа Tailark marketing blocks), которые могут работать лучше для landing pages. Это тоже нужно проверить.

## Новое открытие: Aura.build

Последняя гипотеза перед финальным решением. Есть сервис Aura.build от @MengTo (создатель Design+Code).

Идея: AI генерирует готовые шаблоны landing pages уровня Linear/Vercel. Экспорт в чистый HTML + Tailwind, потом конвертируется в React. После экспорта код полностью твой, нет привязки к сервису.

Почему это интересно: вместо того чтобы строить каждую секцию с нуля, берёшь готовый шаблон и адаптируешь под бренд. В комбинации с Unicorn Studio это может дать результат уровня premium за fraction of time.

Риски: не тестировал. Нужно проверить качество кода и время на адаптацию.

## Другие найденные инструменты

**ShadCN Create** (ui.shadcn.com/themes): кастомизация компонентов shadcn ДО установки. Можно задать цвета, шрифты, радиусы заранее. Решает часть проблем с generic-видом. Может пригодиться для функциональных частей сайта типа форм.

**Drawbridge**: визуальная отладка с Claude Code. Можно указывать на элементы интерфейса и получать фиксы без словесных описаний. Полезно на этапе полировки.
